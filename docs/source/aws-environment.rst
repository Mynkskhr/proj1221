AWS Environment Ownership – Scope and Responsibilities
======================================================

Environment Overview
--------------------
Managed AWS environments supporting production workloads, including:

- Compute: EC2 / container-based services
- Storage: S3
- Databases: RDS
- Identity: IAM
- Observability: CloudWatch

Scale
-----
- Single and/or multi-account environments (where applicable)
- Production-grade workloads with CI/CD pipelines
- Active incident response and operational ownership

Responsibilities
----------------
- Infrastructure provisioning using Terraform
- IAM design and security hardening
- CI/CD pipeline implementation and governance
- Backup and disaster recovery planning and testing
- Monitoring, alerting, and incident response
- Cost awareness and optimisation (where required)

Operational Model
-----------------
- Ownership of production reliability
- On-call support and incident handling
- Continuous improvement of infrastructure and processes

Outcome
-------
- Stable and secure production environments
- Improved operational visibility
- Reduced risk through structured governance
